Historically we use Skype for internal messages at chez moi. The rate of undelivered messages is MUCH higher than it used to be, so we are looking for a replacement. Skype was bought by MSFT a long time ago; and they only care about the Skype business accounts (which from a business perspective makes sense). I have read that MSFT will close its Skype servers at some point (from general news site, not a official MSFT plan, so I'm not bothering with a reference), as they would prefer people to use MSFT teams.
Assumed previous reading, is an architecture level view of SIP + VOIP, P2P networking, encryption architectures.

Software selection metrics

  • Run on ubuntu, android, and supports win32
  • Support some type of messaging
  • Support SIP/VOIP
  • Doesn't eat all battery power, especially when not in active use

Nice to haves

  • Prefer P2P architecture, but that is not compulsory
  • Message edit capacity

Candidates

Not a candidate, but here is a huge list of VOIP software at 1.

  • Blink project 2 under GNU GPL3 licence. The 3 project has support for SIP, VOIP, IM, MSRP, VNC and XCAP. It claims to have OTR and ZRTP security protocols. The more commercial side is called blink voice 4 5. QT version supports linux, win32; and macs with the older py3/cocoa build. Improved install instructions are at the end on this page; standard install instructions are 6. Doesn't mention message editing at 7
  • linphone project 8 available under GNU GPLv3, git. The wiki 9 says it supports SIP, SIMPLE, RTP/RTCP, and security wise SRTP ZRTP. The wiki doesn't mention messaging, but 10 does. There is a summary architecture diagram at 11. Install instructions for ubuntu are 12 13
  • empathy 14 licence GNU GPL, started life as a chat client but also now supports VOIP. As what was previously a core piece of Ubuntu software, Empathy is very easy to install (package name is empathy). Pls note empathy project owners say that encryption is outside of scope for them. Secondly, the original group dev gnome group are not doing active development.
  • teamspeak 15 is closed source, with a freemuimum category. The install process is manky 16 newer=better 17. There are *run files available 18

UPDATES

I wrote that at the start of the lockdown, Zoom hadn't gotten this far yet etc etc. In 2022, for my many remote meetings I use one of (this section doesn't mention protocols at all; and is necessary to be current, but is really dull):

  • Zoom 19 20 and it also available via apt, has a chat client, but I don't use it much. Has reasonable video quality and connection stability. Meetings are made via the website 21. You need to manage meeting length, or use a paid account; but controlling meeting length is good practice anyway. [This is a commercial product.]
  • slack (via chrome 22, as its lighter than the electron app). I was using this chat client a lot at Hoptroff. It also has voice and video streaming 23. The electron app has lockups, chrome doesn't. [This is a commercial product with a reasonable freemium.]
  • microsoft teams, yes on linux, has good quality outputs. I think “teams-for-linux” rather than “teams-insider”. Logging in is a overweight monstrosity, as it wants to do everything via outlook. I haven't had a situation to use the chat client very much, as the sales and BD teams weren't chatting. It wants to update about once per week, some notes say there is an electron version and a native version; where the native version has more updates 24. Installed via adding a new apt server; or a direct download from microsoft website 25 26 27. [This is a commercial product, and needs a paid account to use.]
  • signal (installed to desktop and phone via respective app shops). Good phone integration. Desktop version doesn't cope with dropped networks perfectly; and to support to data privacy flushes messages when this happens. Has many icon packs. I was using this in 2020... I think this is opensource, I haven't paid anything to use it (but they have many servers...)
  • viber (on phone, install via app shop) my other half uses this to talk with relatives in “the old country”. Has many icon packs. [This is a commercial telephony product with the same structure as Skype]
  • telegram (on phone, install via app shop) I use this sometimes. Not used VOIP here, but I was using this in 2020...
  • whatsapp (on phone, or ubuntu via hacks 28 ). The link provides a list of solutions in varying levels of maturity or hackiness. This app is owned by Facebook 29. I have used it.
