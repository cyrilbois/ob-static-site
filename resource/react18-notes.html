Basic component API, react18

The newest version of react has a slightly different life cycle model; and some of the older process has been retired 2 3 4.

React18 hooks 5

  • A vastly over engineered diagram, drawn in React 6
  • standard: useState() 7, useMemo(), useReducer(), useEffect() 8 which everyone must have used.
  • componentDidMount() 9
  • componentWillUnmount() 10
  • shouldComponentUpdate() 11 12 used as a test, and is a micro-optimisation, shouldn't affect local state
  • componentDidUpdate() 13
  • getSnapshotBeforeUpdate() 14
  • static getDerivedStateFromProps() 15 check if needed first as this will make code more complex.
  • constructor() 16, only used when your components are more complex
  • static getDerivedStateFromError() 17
  • componentDidCatch() 18
  • render() 19
  • useTransition(), (=> isPending, startTransition() ) ~ act as replacement/ improvement for useEffect() 20 21.
  • userId() 22 ~ added for server side rendering, generate an ID
  • useDeferredValue() ~ have an current value until a Promise resolves then use the new value.
  • useSyncExternalStore() 23 ~ probably for library dev only,
  • useInsertionEffect() 24 ~ a new hook that allows CSS-in-JS libraries
  • useLayoutEffect 25
  • useCallback
  • useRef
  • useImperativeHandle
  • useDebugValue

React18 functions (which are not hooks)

  • createRef()
  • setState()
  • lazy() for SSR
  • unmount()
  • createRoot() 26, which is in react-dom project
  • forceUpdate()
  • flushSync() ~ most updates are async and batched; this makes updates synchronous

React18 builtin tags

  • Suspense, used to FP style supply a loading screen without branching, needs createRoot() to be active 27
  • SuspenseList 28
  • Offscreen
  • StrictMode 29 Note only for dev builds

Things some guy on twitter didn't want people using 30

  • Class components ~ Use react-error-boundary package
  • useEffect() ~ often overused, there are options
  • Context ~ use for mostly static global config; can also use props and or third party libraries
  • prop-types, he says its “only at runtime”, and use typescript types; IMO use both
  • style - Try: Tailwind, CSS modules, --CSS-in-JS--, Sprinkles
  • create-react-app - Try: Vite, Next, Remix, Astro, Gatby, Docusaurus, Redwood, NX, Hydrogen
