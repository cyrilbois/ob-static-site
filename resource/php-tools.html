As Composer is more mature, I am trying to drop Pear entirely.

In my last contract this took too long to setup, so I am writing myself a cheat sheet. Yes I expect the URLs to get out-of-date quite fast.
If I compress this down to a set of GIT URLs and a composer file, I can start work on the speed of the network connection. If I can bundle this, even when tools aren't needed, there is no delays when time scales start to matter.

Standard operating system:

  • Your interpreter:
    • ms
    • mac
    • linux: use normal procedure
    • don't forget all the language extensions (previously resolved via macports)
  • composer
  • if absent GIT or GIT
  • or if necessary SVN beanstalk
  • vim ~ I tend to script large scale changes in this, as its faster. I also use `ed`, when faster processing is needed.
    • macvim
  • eclipse with phpelclipse
  • netbeans with plugins ~ git
  • sublime with plugins
  • if MAC term2
  • if MAC create apps
  • if MAC xcode
  • if MAC macports
  • if MAC homebrew

Webrowsers

vim plugins

  • matchit
  • phpdoc
  • surround
  • TOhtml $VIMRUNTIME/syntax/2html.vim
  • . . . . add more here

Standard PHP tools:

  • phpunit:
    • see composer
    • linux: use normal or pear
    • mac: read around, use pear
    • ms: install linux start again
  • phpdoc
    • pear channel-discover pear.phpdoc.org; pear install phpdoc/phpDocumentor
    • see composer
  • phpunit
    • pear install phpunit/PHPUnit
    • see composer
  • add the skel-gen
    • pear install phpunit/PHPUnit_SkeletonGenerator
    • wget https://phar.phpunit.de/phpunit-skelgen.phar
  • phpcs
    • see composer
    • pear install PHP_CodeSniffer
  • phploc
    • see composer
    • wget https://phar.phpunit.de/phploc.phar
  • PHP depend
    • composer “pdepend/pdepend” : "@stable”
