Concise intro (links mostly)

SVG aka Scalable Vector Graphics 1 is a nice image format for multiple screen sizes, as is its vector 2; secondly it's XML, so hackable from JS examples, examples of capacity from JS, a real world map, and another. In 2017 all browsers support SVG without awkward tech problems 3. SVG is better than HTML5 Canvas, as it supports events better. There is more discussion 4 5. SVG images are seen by the browser as a small subdocument, with an editable DOM 6. As SVG is a w3C invention, a good source of spec news is of course w3c 7 8 . By 2023, there have been no updates to the SVG spec since SVG2 in 2018 9.

However this boring info that can be gained from any wordpress techblog is dull, and easy. Pls refer to the examples link if you are after a visual inspiration.

Reading this far will improve comprehension on SVG.

SVG structure

The following section is fitting SVG into the structures/ architecture of other graphics systems.

  • SVG colours are encoded in sRGB 29 (relationship between these 30). sRGB is supposed by be “the simple solution, acting like a windows computer”.
  • SVG uses a similar but different CSS instructions, and has an Animate element 31 32. SVG has quite good font support 33. I have seen images of text stored as SVG quite a few times, so at least one popular image editor can export text to SVG (but not SVG fonts which are different).
  • It can be used as a 2D or 3D 34 35 36 modelling system, SVG needs several co-ordinate notations. Summarising viciously, to use SVG, define polygons in a set of [I suppose document] co-ordinates 37, apply any transforms 38 or clipping 39 for artistic direction; then apply to screen co-ordinates to be able to define pixels. Here is a guide for people who have studied less maths 40 41 42. The links to o'Reilly [publisher] book chapters previously where on it's blog and open. I will swap the links to ones without a paywall.
  • Poorly organised SVG generators will lead to excessive file size volumes. Solutions obvious to webdev are listed in svg-optimisation 43 44. I reference git as a focussed reference, or svg-optimiser.
  • SVG supports several different methods for representing curves, TinySVG (for mobile) supports Bézier and elliptical curves 45, the SVG spec mentions quadratic and cubic Bézier curves 46 in addition.
  • As part of drawing for 3D, or just stacked polygons, the SVG spec has depth sorting. SVG2 47 is tightly related to CSS2 z-index. SVG1 48 defines many custom attributes which are not in CSS.
  • As a rendering toolkit spec, rather than a image; SVG includes some lighting models 49 phong 50 51 52
  • An academic wrote a list of 'primitives' rendering features of SVG animation 53 (http only), he also has a book from 2012 54. The maximum number of polygons is quoted at around 1000 ~ 3000 55, which is really low for actual rendering engines.

IMPORTANT: That limit should be checked on phones. In my personal experience, a sales guy added a high quality photo stored as a SVG to the login page on an app he was selling; it took 5s-10s to load the login page on my work laptop at that point, as it was busy rendering the SVG.

  • With current browsers, SVG respects CORS limits same as other assets 56.

SVG can be used for Favico, but this is not supported by Safari 57 favicon-notes. In theory SVG animation could be used in Favico, but again Safari support is important to many businesses.
