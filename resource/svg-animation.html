If you somehow do not know anything about SVG, I suggest skimming the format notes first.

SVG Format notes SVG Editors SVG Optimisation SVG Animation

SVG animation should be a replacement for the now obsolete 'flash animation'. It has similar features (compare 1 2 and 3 - nb this is a spec), ignoring out of scope items such as playing sound. As the tech platform is directly tied to JS, additional features like websockets are easily accessible. The spec for SVG supports some animation 4. As text encoded formats, JS + SVG are better for MIT style 'hacker'/ graphics hack cultures. Not being flash means that there are many security and update problems that are irrelevant. Javascript is strictly designed to allow arbitrary people to run code on each client PCdevice (see apps on phones...). SVG + JS have a different focus to GIF animation. Currently GIFs are mostly used for downsampled video clips; which is not a good deliverable for SVG + JS.
SVG spec includes Animate tag 5, AnimateMotion? 6 and AnimateTransform? 7. I strongly advise careful testing before deployment to production; as the supported details are not well documented.

Animation via CSS

Obviously SVG can be animated with CSS3 8 9. Please note that modern CSS includes Trig calculations 10 11, combined with calc 12 and CSS variables implies the requirement for webGL, or access to OpenGL is diminished. CSS is supposed to be a declarative language, although that last sentence is procedural approach.
CSS supports 'animation' as a verb 13; and practically CSS class names would need to be applied by manually editing the SVG file, but that should be easy for the average developer. It is more practical to use an animation framework to ensure consistency across different devices. There are many animation micro-frameworks/ libraries/ CSS-code generators, there are many listed 14 from 2021. No project in NPM tries to claim that combination of keywords 15, but they do for JS animation. The best match seems to be 16.

6 animation libraries via JS

There are many resources on animation via JS/TS. As an imperative programming language it is fairly simple to apply changes across time, a low bar to entry.

  • The best and well-known library is d3, AFAIK. I have used D3, it feels like OpenGL. Sources: docs git d3 in NPM. As a widely known tool, this has books 22 23 24 25 26, courses 27, manuals 28 29 30 31 etc. This is a complete toolkit, and supplies many features; it's terminology closely matches W3C specs; the only downside is that it is not small, and has a learning curve. It has a twitter channel 32. EXAMPLE: A fancy type of diagram that can be achieved with d3, but not most other tools 33. It is quite easy to write a book in D3; it also can do animation 34 35 36, of course, an example for React 37 and also many interactivity options 38
  • Vega in NPM as vega, released under BSD3 clause, docs git. The tool either outputs Canvas or SVG, depending on config. There is a second project vega-lite, which is supposed to have a higher level of abstraction 39. 40.
  • SVGjs as mentioned on the editors article, supports animations 41. It supports a full range of actions, but the data is defined by a different step to these functions 42.
  • Paperjs git docs installed from NPM as paper, licensed as MIT. It has support packages for Node, which uses Cairo and Pango libraries. They announce that they test with Qunit. They have a twitter handle 43. Animation features 44.
  • vivus git released under MIT, is a narrow small SVG animation library. “This is designed to be small and stand-alone, without dependencies” (quote from NPM docs) and can be installed via NPM vivus. Read the animation notes on NPMJS to see if this meets your requirements; it doesn't have support for text animation (via text elements), and fills will come-up wrong. There is an extension called react-vivus, if you are using an old version of React.
  • jquery-drawsvg git NPM with a MIT license, is a tool to slowly render an SVG, as an animation 45. The speed is intensional, and wraps jQuery.animate() 46 ~ hence the name. It doesn't include features to change an SVG. I can only see this useful if you are using jQuery.
