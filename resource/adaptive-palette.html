I did a crawl on the internet/ npmjs, I cannot find a CSS framework that advertises adaptive colours. I thought it a good idea to replace the style sheet as its old and accumulated code; the newer code is smaller. If I add more usability features (e.g. dark-mode), the smaller style sheet will be necessary. This palette is actually the first CSS palette I have made, I normally use the premix colours in the framework, or extract values from the style-guide. However the adaption features ares a hobbyists over-engineered solution; if you tell your browser that you a prefer an extreme on prefers-contrast, the palette will match it. It would be cool if there was a thing for “I'm in an oversaturated environment”, and put the brightness up; but I do not have access to that data programmatically.

Please see accessible-css for more notes on colour adaption. This code is written with RGBA colours. There are other colour spaces available, but this has best support in browsers and external tooling.
