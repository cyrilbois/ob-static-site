  • Only read if keen; I know pages about the site you are reading are very boring.
  • Some of the early changes where a learning curve for me, but none of the ones after 2014 are. I am adding more features to match the current scale of the data.
  • This site has always been a UTF-8 site, this was more rare when I started it.
  • The articles on this site reflect semantic-web concepts from 2004 era; they each have a meta header used to build the HTML head and for my custom indexing.
  • One of the section headings on this homepage was “everyone is as good as their sources”. I removed that section; however you can see all my sources (all those Harvard style reference links, all of them. Mean average of 30 or-so per article).
  • In 2015, I added font-awesome for logos making the site load faster; in 2020, I replaced font-awesome with fontello as it's smaller again.
  • In 2024, I recompressed my compressed WOFF2 fonts, as the place I had downloaded them from hadn't compressed properly. #leSigh, its just gzip.
  • As random stats, I built a second index for discoverability, based in article name (used in the URL). There only 4 widely used initial letters (): C..., J..., P..., and S... This is an example of why I think alphabetical indexing is poor UX. I defined 'widely used' to be more than 5% of articles, as I don't think I have more than 15 articles on the same same topic, so that implies the letter is used by multiple topics.
    • This is the reason that github suggests random names.
  • In 2017, I added the create date and edit date meta-data. The last edit date feature is less useful than I would like (in 2020, then again end of 2022); as I needed to edit every single existing article to add more meta data.
  • In 2018 I put articles in article groups to improve discoverability of content.
  • If I had spent one more day on site infrastructure in 2013, this would have been a JAMstack site since then. UPDATE 2022: my webhost was bought; so I have done this.
  • In the end of 2022, I updated many references to be HTTPS. I think need a process to rebuild all the references every year at the least often, to update dead links. I now have a link report on the end my site build script #leSigh.
  • In 2012 I made the site mobile capable (template v3); but I wasn't expecting anyone to use it from a phone. I put text in columns for readability, like newspapers. I added “page chunks” to the articles so users didn't need to scroll up and down all the time (template v4). I added an RSS, and a site-map in that update as the content had increased. Then 2022, I updated the HTML to a more modern version, with better accessibility (template v6).
  • I am replacing the 2010 and 2012 era CSS with a BEM CSS design; again so I can improve adaptability to different peoples preferences and machines. My palette is now 9 colours, rather than erm, about 50. I will tidyup the font declarations next.
  • I have dropped my “manual mouse hover” code (JS used for triggering adjacent elements), as CSS v4 is now widely enough supported. In 2024 I removed all remaining shim for MSIE.
  • In order to be semantic, I have standardised that code samples are wrapped in "<CODE lang=“XXX”><PRE class=“code " lang=“XXX”>" blocks. The HTML spec says that PRE isn't for programming languages, but using PRE makes it much more readable.
  • I am tagging older articles with a warning in the strap-line area. I would like to add an URL map (the Apache feature) to pull in likely misspellings, and to make everything lower case
  • It would be nice to drop the jQuery for Alpine. I can probably make the code shorter with more modern structures.
  • A marketing person ~ after the have gotten over a shock about “no wordpress found” ~ will logically ask wouldn't it be faster to use a common share library e.g. 1 2, rather hand-rolling one. Hopefully they then realise a JAMstack site with strong PWA doesn't really work like that. This does help me understand those media companies API better.
  • After 2012, many of the articles are where I needed to be a local expert in XYZ in a hurry. I am trying to be planned and organised about adding more content. I have deleted 20% of my content that I do not think is useful today.
  • You will note all the github links magically get the Octocat logo. I also decorated the docs links. I do not have a NPM logo to repeat the same trick.
  • This git-repo / this project has no tests. The engine that creates the HTML does have test-scripts, and this was the biggest test population I made at that point, as I needed to test a language; not an app, report or something.   • Currently my the directory of raw content text, which hasn't been compiled to HTML weighs abit bigger than 3MB. I have about 240 articles, as I deleted the oldest 20% in Jan 2022.   • I have moved some older JS based “UX and information retrieval improvements” to being built in CSS.
